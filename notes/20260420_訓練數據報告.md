## 一、資料集規模

| 測試集（cnews.test.txt） | 2,912 筆 | 1.2 MB |

### 文字長度（標題 / 摘要欄）

| 統計項 | 數值 |

| 中位數 | 93 字元 |

> 中位數 93 遠低於平均 172，代表大多數陳情文字偏短，少數長文拉高了平均值。

> 這是臺東市公所、國際發展及計畫處 F1 偏低的主因之一，後續優化建議採用 Class Weighting。

## 三、訓練過程與時間

### TextCNN

| 項目 | 數值 |
|------|------|
| 設定最大 Epochs | 30 |
| 實際訓練 Epochs | 18（Early Stop，patience=5） |
| 每 Epoch 平均耗時 | ~2 秒 |
| **總訓練時間** | **約 40 秒** |
| 最佳驗證準確度 | 70.2%（epoch 13） |

各 Epoch 進展：

```
Epoch 001: val_acc=0.5953
Epoch 003: val_acc=0.6389
Epoch 007: val_acc=0.6999  ← 明顯跳升
Epoch 010: val_acc=0.7010
Epoch 013: val_acc=0.7016  ← 最佳
Epoch 018: Early Stop（連續 5 epochs 無改善）
```

### BERT（hfl/chinese-roberta-wwm-ext）

| 項目 | 數值 |
|------|------|
| 設定 Epochs | 5 |
| 每 Epoch 平均耗時 | ~218 秒（約 3.6 分鐘） |
| **總訓練時間** | **約 18 分鐘（1,092 秒）** |
| 訓練吞吐量 | 106.8 samples/sec |
| 最佳驗證準確度 | **76.2%（epoch 1）** |

各 Epoch 評估結果：

```
Epoch 1: eval_accuracy=0.7617, eval_loss=0.749  ← 最佳
Epoch 2: eval_accuracy=0.7438, eval_loss=0.871
Epoch 3: eval_accuracy=0.7356, eval_loss=1.030
Epoch 4: eval_accuracy=0.7442, eval_loss=1.085
Epoch 5: eval_accuracy=0.7469, eval_loss=1.198
```

> **重要觀察**：BERT 在 epoch 1 就達到最高準確度，之後略微下降（輕微過擬合）。
> 實務上若重新訓練，只需跑 **1～2 epochs** 即可，訓練時間可縮短至 **4～8 分鐘**。

## 四、最終模型評估

| 指標 | TextCNN | BERT | 改善幅度 |
|------|---------|------|---------|
| 測試集整體準確度 | 62.0% | 76.2%（驗證集） | +14% |
| Macro F1 | 0.58 | 0.73 | +0.15 |
| 模型檔案大小 | ~4 MB | ~400 MB | — |
| 推論速度（API） | 極快 | < 1 秒/筆 | — |

### 各局處 F1（測試集/驗證集）

| 局處 | TextCNN F1 | BERT F1 |
|------|-----------|---------|
| 交通及觀光發展處 | 0.67 | 0.77 |
| 國際發展及計畫處 | 0.53 | 0.65 |
| 建設處 | 0.64 | 0.72 |
| 教育處 | 0.65 | 0.79 |
| 環境保護局 | 0.59 | 0.73 |
| 社會處 | 0.70 | 0.86 |
| 臺東市公所 | 0.28 | 0.55 |
| 警察局 | 0.66 | 0.84 |
| 農業處 | 0.51 | 0.71 |

> 臺東市公所為兩個模型的共同弱點（BERT recall 僅 42%），推測原因為訓練資料中語意邊界模糊（業務重疊其他局處）。

---

## 五、後續優化方向

1. **Class Weighting**：對少數類別（國際發展、社會處、農業處）加大 loss 權重，改善 recall
2. **資料增補**：臺東市公所增補至 2,000～3,000 筆可顯著改善
3. **縮短重訓週期**：BERT 僅需 1 epoch（約 4 分鐘）即可達到最佳效果
